    **Exports**
    - `export type YutThrowScenePhase = 'idle' | 'launching' | 'settling' | 'revealing' | 'disposed'`
    - `export interface YutThrowSceneController { startThrow(result: ThrowResult): Promise<void>; resize(): void; dispose(): void }`
    - `export function createYutThrowScene(params: { canvas: HTMLCanvasElement; onPhaseChange?: (phase: YutThrowScenePhase) => void; onReveal?: (result: ThrowResult) => void }): YutThrowSceneController`

    **Renderer and camera**
    - Instantiate `new WebGLRenderer({ canvas, alpha: true, antialias: false, powerPreference: 'high-performance', preserveDrawingBuffer: false })`
    - Create `PerspectiveCamera(35, 1, 0.1, 40)`
    - Set camera position to `(0, 6.5, 8.5)` and `lookAt(0, 0.8, 0)`
    - If `renderer.compileAsync` exists, call it once after initial scene setup

    **Responsive sizing**
    - Implement `resize()` using `canvas.clientWidth * window.devicePixelRatio` and `canvas.clientHeight * window.devicePixelRatio`
    - Cap internal pixel count at `1280 * 720`
    - Call `renderer.setSize(width, height, false)` and update `camera.aspect`

    **Scene styling**
    - Add warm ambient light plus one directional light
    - Create a floor mesh using a cream/warm color close to `#FFFDE7`
    - Give sticks warm wood-toned materials; do not use metallic or photorealistic shading

    **Physics world**
    - Create `new World({ gravity: new Vec3(0, -9.82, 0), allowSleep: true })`
    - Add one static ground body aligned with the visual floor
    - Create four dynamic stick bodies with:
      - `mass: 0.9`
      - `allowSleep: true`
      - `linearDamping: 0.31`
      - `angularDamping: 0.4`
      - `sleepSpeedLimit: 0.15`
      - `sleepTimeLimit: 0.6`

    **Stick spawn layout**
    - Spawn stick slots at x positions `[-1.8, -0.6, 0.6, 1.8]`
    - Initial y range: `1.8` to `2.4`
    - Initial z range: `0.2` to `0.8`

    **Throw launch**
    - `startThrow(result)` resets body transforms, sets phase to `'launching'`, derives target poses from `buildTargetPoses(result)`, and applies these exact impulse/torque seeds by slot:
      - slot 0: impulse `(-0.8, 5.2, -1.6)`, torque `(-5, 3, 2)`
      - slot 1: impulse `(-0.25, 5.4, -1.3)`, torque `(4, -2, 3)`
      - slot 2: impulse `(0.25, 5.1, -1.4)`, torque `(-3, 4, -2)`
      - slot 3: impulse `(0.8, 5.3, -1.2)`, torque `(5, 2, -3)`

    **Animation loop**
    - Use `requestAnimationFrame`
    - Step cannon with `world.fixedStep(1 / 60, 10)`
    - Copy each body `position` and `quaternion` into its corresponding mesh every frame
    - After launch, switch to `'settling'`
    - Treat the throw as settled when `world.hasActiveBodies === false` for `6` consecutive frames, or when elapsed throw time reaches `2200ms`

    **Final correction**
    - After settle, tween sticks over `180ms` to the target poses from `buildTargetPoses(result)`
    - Face mapping rule:
      - `flat` => final local x rotation `0`
      - `round` => final local x rotation `Math.PI`
      - always preserve the target `yaw` from `buildTargetPoses`
    - Then set phase to `'revealing'` and call `onReveal(result)`

    **Cleanup**
    - Store the RAF id and any window resize listener in closure state
    - `dispose()` must cancel RAF, remove the resize listener, remove every dynamic body and the ground body from the world, call the resource tracker disposal, call `renderer.dispose()`, then `renderer.forceContextLoss()`, and finally set phase to `'disposed'`

    Add comments only above the settle-detection branch and the final-correction branch because those are the non-obvious parts.
