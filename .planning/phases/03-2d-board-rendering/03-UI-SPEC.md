# Phase 3 — UI Design Contract

> Visual and interaction contract for the 2D SVG Yut Nori board. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none |
| Preset | not applicable |
| Component library | none (native SVG + React) |
| Icon library | none (all visuals are SVG primitives) |
| Font | Geist (via next/font/google, already configured) |

**Rationale:** This phase renders an interactive game board using native SVG elements animated with motion/react. No form-based UI component library is needed. All visual elements are SVG circles, lines, paths, and text rendered directly in React.

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Gap between stack badge and token edge |
| sm | 8px | Offset between overlapping pieces at same station |
| md | 16px | Padding between board SVG edge and container |
| lg | 24px | Gap between board area and info bar below |
| xl | 32px | Internal SVG padding (station coords stay within 40-460 in 500px viewBox) |
| 2xl | 48px | Not used this phase |
| 3xl | 64px | Not used this phase |

Exceptions: SVG internal coordinates use a 500x500 viewBox with its own coordinate system. The spacing scale above applies to the HTML container layout around the SVG, not to positions within the SVG. Inside the SVG, station spacing is determined by the diamond geometry (see SVG Coordinate System section).

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 14px | 400 (regular) | 1.5 |
| Label | 12px | 600 (semibold) | 1.2 |
| Heading | 20px | 600 (semibold) | 1.2 |

**Usage in this phase:**
- **Body:** Not used in board SVG; part of the project-wide type scale for future info panel text
- **Label:** Stack count badge text inside SVG (rendered as SVG `<text>` at fontSize 10 in viewBox units, which maps to ~12px at typical display size). Also used for HOME zone piece count labels.
- **Heading:** Throw result display text (e.g., "do", "yut") shown above or beside the board

**SVG text sizing note:** SVG text uses viewBox-relative units. At a 500px viewBox displayed in a ~375px mobile container, 1 viewBox unit = ~0.75 physical px. Stack badge text at fontSize 10 (viewBox) = ~7.5px physical, which is acceptable for a 1-2 digit number badge. If readability is an issue, increase to fontSize 12.

## Animation Contract

### Hop Animation (D-07)

| Property | Value |
|----------|-------|
| Duration per hop | 200ms |
| Easing (intermediate hops) | `[0.25, 0.1, 0.25, 1.0]` (ease-out cubic bezier) |
| Easing (final landing) | `spring { stiffness: 400, damping: 20 }` |
| Total for do (1 step) | ~200ms |
| Total for mo (5 steps) | ~1000ms |

## Interaction Contract

### Piece Selection Flow

1. Player's turn begins; pieces with valid moves get glow ring animation
2. Player taps a piece token (minimum 44px hit area via invisible circle r=22)
3. Selected piece shows gold selection ring; glow ring stops on other pieces
4. Valid destination stations show pulsing highlight dots
5. If destination is a branch point (S5/S10), two destinations highlight in different colors (gold = continue outer, green = take shortcut)
6. Player taps a destination dot to confirm move
7. Hop animation plays along intermediate stations
8. After animation, piece appears at final position

### Piece Deselection

- Tapping a different selectable piece switches selection
- Tapping the already-selected piece deselects it (removes highlights)
- Tapping empty board area deselects

### HOME Zone Interaction

- HOME pieces displayed below the board in team-colored zones
- HOME pieces are selectable when the player has pending throw results
- Tapping a HOME piece highlights the entry destination (station 0 + steps)

### State Separation (Visual vs Logical)

During hop animation, the logical game state updates immediately to the final position. The visual layer uses a separate "animating piece" overlay that plays the hop sequence. After animation completes, the overlay is removed and the piece renders at its logical position. An `isAnimating` boolean flag prevents interaction during animation.

## Layout Contract

### Mobile-First Vertical Layout (D-03)

```
+---------------------------+
|                           |
|     Board SVG Area        |   60-70% of viewport height
|     (aspect-square)       |
|                           |
+---------------------------+
|  [Player HOME] [AI HOME]  |  HOME zone: team pieces off-board
+---------------------------+
|                           |
|    Info / Controls Area   |   30-40% remaining (Phase 4/5)
|                           |
+---------------------------+
```

| Zone | Height | Content |
|------|--------|---------|
| Board area | `flex-1` up to 70vh | SVG board with `max-w-[500px] mx-auto` |
| HOME zone | 48px | Two team zones side by side, tokens + count |
| Info area | remaining | Reserved for Phase 4 throw button and Phase 5 turn UI |

### SVG Layer Order (z-index via render order)

1. **Background:** Board fill (cream), diamond outline, edge lines, diagonal lines
2. **Stations:** 29 station dots
3. **Highlights:** Pulsing destination indicators (only when piece selected)
4. **Pieces:** Team tokens at their station positions
5. **Animating piece:** Single piece overlay during hop (topmost, above all other pieces)

## Accessibility

| Concern | Approach |
|---------|----------|
| Touch targets | Invisible r=22 circle behind each token ensures 44px+ touch area on mobile |
| Color contrast | Team tokens (blue/red) on cream board exceed 3:1 contrast ratio. Station dots (dark brown on cream) exceed 4.5:1. |
| Color-blind safety | Player vs AI distinguished by blue vs red (safe for most forms of color blindness). Additional shape differentiation deferred to Phase 6 character pieces. |
| Screen readers | SVG root has `role="img"` and `aria-label="윷놀이 게임판"`. Individual interactive elements have `aria-label` describing action. |
| Reduced motion | Respect `prefers-reduced-motion`: skip hop animation (instant position), disable pulse (static highlights) |
| Keyboard | Not required for mobile WebView target. Deferred. |
