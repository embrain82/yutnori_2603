    Exact behavior rules:
    - `startGame()` uses `createInitialGameState()` and `createSession()`, sets `phase: 'readyToThrow'`, and preserves `couponConfig`

      - sets `activeThrow = result`
      - sets `phase = 'throwing'`
    - `finishThrowReveal()`:
      - clears `activeThrow`
      - if `turnState.throwsRemaining > 0`, return to `'readyToThrow'` for player or `'aiThinking'` for AI
      - otherwise consume queued results FIFO until a result with at least one move candidate is found
      - store that result in `activeMove`, store candidates in `moveCandidates`, and set `phase = 'selectingPiece'`
      - if all queued moves are impossible and nothing remains, switch turn to the other team using `createTurnState(nextTeam)` and phase `'aiThinking'` for AI or `'readyToThrow'` for player

    - `selectDestination(stationId)` resolves the matching candidate, applies `applyMove`, stores the updated `pieces`, captures `pendingStack` when offered, and sets `pendingAnimation`
    - if `applyMove` captured pieces, increment `turnState.throwsRemaining` by `1`
    - `resolveStack(true)` calls `confirmStack`; `resolveStack(false)` calls `declineStack()` and leaves pieces untouched
    - `completeAnimation()` checks `checkWinCondition(pieces)` first, then either:
      - enters `'confirmingStack'` when `pendingStack` exists
      - re-enters move selection if queued results remain
      - re-enters throwing if `throwsRemaining > 0`
      - or switches to the other team
