  <action>
    Implement all 7 functions in src/lib/yut/capture.ts. Follow pure-functional patterns (no mutation, return new arrays/objects).

    **detectCapture(pieces, movingPieceId, destinationStation):**
    - Guard: if destinationStation is HOME (-1) or FINISH (-2), return { captured: false, capturedPieceIds: [], grantExtraThrow: false }
    - Find the moving piece's team
    - Filter pieces at destinationStation where team !== movingPiece.team AND station >= 0
    - If opponents found: collect all their IDs (including stackedPieceIds of any leader), return { captured: true, capturedPieceIds: [...], grantExtraThrow: true }
    - Per D-01: grantExtraThrow is always true when capture happens

    **executeCapture(pieces, capturedPieceIds):**
    - For each capturedPieceId, set position to { station: HOME, routeId: '', routeIndex: -1 }
    - Clear stackedPieceIds to [] and stackedWith to null for each captured piece (per D-02)
    - Return new pieces array

    **detectStack(pieces, movingPieceId, destinationStation):**
    - Guard: if destinationStation is HOME (-1) or FINISH (-2), return { canStack: false, targetPieceId: null }
    - Find moving piece's team
    - Filter pieces at destinationStation where team === movingPiece.team AND id !== movingPieceId AND stackedWith === null (only independent pieces or leaders, not followers)
    - If friendly found: return { canStack: true, targetPieceId: friendly.id } (first match)
    - Otherwise: { canStack: false, targetPieceId: null }

    **confirmStack(pieces, arrivingPieceId, existingPieceId):**
    - existingPiece becomes/stays leader
    - If arrivingPiece has followers (stackedPieceIds.length > 0): transfer all followers to existingPiece
    - Add arrivingPieceId to existingPiece.stackedPieceIds
    - Set arrivingPiece.stackedWith = existingPieceId
    - Set all transferred followers' stackedWith = existingPieceId
    - Clear arrivingPiece.stackedPieceIds = []
    - Return new pieces array

    **declineStack():**
    - No-op function. Per D-05, declining means pieces coexist independently. No state change needed.

    **moveStackGroup(pieces, leaderId, newPosition):**
    - Find leader piece, update its position to newPosition
    - For each follower in leader.stackedPieceIds, update their position to newPosition
    - Return new pieces array

    **applyMove(pieces, pieceId, moveResult):**
    - This is the orchestration function.
    - Step 1: Update moving piece's position to moveResult.newPosition (if finished, set FINISH sentinel)
    - Step 2: If piece is a leader (has stackedPieceIds), call moveStackGroup for followers too
    - Step 3: If piece finished (moveResult.finished), skip capture/stack -> return with no interactions
    - Step 4: detectCapture at destination station
    - Step 5: If captured, executeCapture on the pieces
    - Step 6: detectStack at destination station (AFTER capture per Pitfall 2)
    - Step 7: Return MoveOutcome with updated pieces, capture result, and stack opportunity

    CRITICAL ORDERING per Pitfall 2: capture FIRST, then stack detection. This ensures opponent is removed before checking for friendly stacking.
